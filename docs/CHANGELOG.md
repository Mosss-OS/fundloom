# Changelog

All notable changes to Fundloom will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]
### Added
- Streaming donations functionality with subscription management
- Comprehensive documentation suite (privacy, terms, security, etc.)
- AI integration via secure Supabase Edge Functions
- Real-time updates using Supabase Realtime
- Localization documentation and framework
- Enhanced dispute resolution UI

### Changed
- Updated Factory contract address to use new GovernanceToken
- Improved dispute voting UI with better feedback
- Added proper ownership controls to GovernanceToken

### Fixed
- Resolved compiler issues with stack too deep in dispute functions
- Fixed enum references in test files
- Updated deployment scripts

## [v1.0.0] - 2026-04-26
### Added
- Initial release of Fundloom platform on Base Sepolia testnet
- Smart contract factory with milestone-based escrow
- Campaign creation, funding, and withdrawal functionality
- Dispute resolution and appeal system
- Governance token for DAO voting
- AI features: campaign optimizer, fraud detection, donor matching
- Real-time updates for campaign data
- Responsive frontend with React and Tailwind CSS
- Wallet authentication via Privy
- Supabase backend for user data and activity tracking

## [v0.1.0] - 2026-03-15
### Added
- Project initialization
- Basic smart contract structure
- Initial frontend layout
- Basic documentation outline
