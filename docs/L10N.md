# Localization (L10n) and Internationalization (I18n)

Last updated: April 2026

## Overview

Fundloom is designed to be a global platform supporting multiple languages and regional formats. This document outlines our approach to localization and internationalization.

## Supported Languages

We plan to support the following languages in order of priority:

1. English (en) - Primary language
2. Spanish (es)
3. French (fr)
4. Portuguese (pt)
5. Chinese (zh)
6. Japanese (ja)
7. Korean (ko)
8. German (de)

## Implementation Strategy

### Translation Files
All user-facing strings will be externalized into JSON translation files located in:
```
/src/locales/
├── en.json
├── es.json
├── fr.json
├── pt.json
├── zh.json
├── ja.json
├── ko.json
└── de.json
```

Each file will contain key-value pairs where:
- Keys are dot-notation strings describing the UI element/context
- Values are the translated strings

Example structure:
```json
{
  "navigation": {
    "home": "Home",
    "explore": "Explore",
    "create": "Create Campaign",
    "dashboard": "Dashboard"
  },
  "campaign": {
    "title": "Campaign Title",
    "description": "Campaign Description",
    "goal_amount": "Funding Goal",
    "deadline": "Campaign Deadline"
  }
}
```

### Framework Integration
We will use `react-i18next` for internationalization in our React application.

### Date and Number Formatting
Dates, times, numbers, and currencies will be formatted according to locale standards using:
- Intl.DateTimeFormat
- Intl.NumberFormat
- Intl.RelativeTimeFormat

### Right-to-Left (RTL) Support
While our initial language set doesn't include RTL languages, we will architect our UI to support RTL layouts for future expansion.

## Translation Workflow

1. **Development**: Developers add new UI strings with English default values
2. **Extraction**: Automated script extracts new/changed strings to translation files
3. **Translation**: Community translators or professional services translate strings
4. **Review**: Translations reviewed by native speakers
5. **Integration**: Translated files merged into codebase

## Contributing Translations

Community members can contribute translations by:
1. Forking the repository
2. Adding or updating translation files in `/src/locales/`
3. Submitting a pull request with their translations
4. Following the existing JSON structure and formatting

## Fallback Mechanism
If a translation is missing for a specific locale, the system will fall back to:
1. English (primary fallback)
2. Then show the translation key in brackets (e.g., `[navigation.home]`) for debugging

## Testing
Localized UI will be tested for:
- Text overflow and truncation issues
- Date/number/currency formatting correctness
- Layout integrity with longer strings
- Proper handling of special characters and unicode

## Future Considerations
- Dynamic language detection based on browser/device settings
- Language persistence in user preferences
- Community-driven translation platform
- Professional translation services for critical content
